Overview of the Flowchart

The flowchart illustrates a data engineering pipeline for algorithmic trading, detailing the journey of data from SFTP and Binance API through extraction, storage, transformation, and trading. Key components include:

  • Data Sources: SFTP for historical CSV files and Binance API for real-time price data.
  • Extraction: Azure Functions pull data from both sources to Azure Blob Storage.
  • Orchestration: Airflow DAGs manage workflows, triggering extraction and transformations.
  • Storage: Snowflake organizes data into raw, curated, and mart layers.
  • Trading: A momentum-based bot uses mart layer analytics for trade execution.
  • Monitoring: Azure Monitor and Snowflake logs ensure pipeline reliability.

This structure ensures seamless data flow and scalability for trading applications.

Flowchart: Step-by-Step Data Flow

The pipeline begins with a trigger and flows through the following steps:

  • Start: An hourly Airflow DAG initiates the pipeline, coordinating all tasks.
  • Data Sources: Two parallel sources—SFTP (CSV files with trade records) and Binance API (real-time BTC/USDT prices)—provide input data.
  • Data Extraction: Azure Functions extract SFTP CSVs and Binance API JSON, storing them in Azure Blob Storage as a staging area.
  • Data Loading: Snowpipe auto-ingests data from Blob Storage into Snowflake’s raw layer, preserving original formats.
  • Data Transformation: Airflow triggers Snowflake tasks to clean data (curated layer) and aggregate it for analytics (mart layer), calculating indicators like EMA.
  • Trading Execution: A trading bot queries the mart layer for EMA signals, executing trades on Binance with a 1:3 risk-reward ratio (0.75% stop-loss, 2.25% take-profit).
  • Monitoring: Azure Monitor and Snowflake audit logs track performance and data access.
  • End: The pipeline completes, and Airflow schedules the next cycle.

Key Components of the Flowchart

The flowchart highlights critical components for a robust trading pipeline:

  • Airflow DAGs: Orchestrate data extraction, loading, and transformation, ensuring automation and reliability.
  • Azure Functions: Handle data pulls from SFTP and Binance API, storing raw data in Blob Storage for scalability.
  • Snowflake Layers: Organize data into raw (unprocessed), curated (cleaned), and mart (analytics-ready) for efficient processing.
  • Snowpipe: Automates data loading from Blob Storage to Snowflake, minimizing latency.
  • Trading Bot: Executes momentum trades based on mart layer analytics, optimizing for 2025’s crypto markets.
  • Security: Azure Key Vault secures credentials, while Snowflake’s role-based access ensures data governance.

Security and Monitoring in the Flowchart

The flowchart incorporates robust security and monitoring:
