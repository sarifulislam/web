Project Architecture Overview

The AI customer support agent uses a modular, scalable architecture:

  • Rasa Chatbot: Handles conversational AI, processing customer queries with NLP-based intents and entities.
  • Python Microservices: Pull data from APIs (e.g., CRM, X) and process interactions for storage.
  • Apache Kafka: Streams real-time customer interaction data for processing.
  • PostgreSQL: Stores data in raw (unprocessed), curated (cleaned), and mart (analytics-ready) schemas.
  • Celery: Orchestrates asynchronous tasks for data processing and chatbot responses.
  • Object Storage: Stores raw JSON/CSV data (e.g., using MinIO or cloud storage like S3/GCS).
  • Monitoring: Uses Prometheus and Grafana for pipeline and chatbot performance monitoring.

This architecture ensures flexibility, scalability, and integration with any cloud or on-premises infrastructure.

Python Microservice: API Data Pull (api_pull.py)


import json
import requests
from kafka import KafkaProducer
from datetime import datetime

def fetch_crm_data():
    try:
        api_endpoint = "https://crm.example.com/api"
        headers = {"Authorization": "Bearer your_api_key"}
        response = requests.get(api_endpoint, headers=headers)
        data = response.json()

        # Send to Kafka
        producer = KafkaProducer(bootstrap_servers='localhost:9092',
                                value_serializer=lambda v: json.dumps(v).encode('utf-8'))
        producer.send('customer_data', data)
        producer.flush()
        return {"status": "success", "message": "Data sent to Kafka"}
    except Exception as e:
        return {"status": "error", "message": str(e)}

Python: Process Rasa Output and Store in PostgreSQL (rasa_processor.py)


import json
import psycopg2
from kafka import KafkaConsumer
from datetime import datetime

def process_rasa_output():
    consumer = KafkaConsumer('rasa_output',
                            bootstrap_servers='localhost:9092',
                            value_deserializer=lambda x: json.loads(x.decode('utf-8')))
    conn = psycopg2.connect(
        dbname="support_db", user="user", password="pass", host="localhost", port="5432"
    )
    cursor = conn.cursor()

    for message in consumer:
        data = message.value
        query = data.get('query')
        response = data.get('response')
        sentiment = data.get('sentiment', 'neutral')  # From NLP model
        cursor.execute("""
            INSERT INTO raw.customer_interactions (timestamp, user_id, query, response, sentiment)
            VALUES (%s, %s, %s, %s, %s)
        """, (datetime.now(), data.get('user_id'), query, response, sentiment))
        conn.commit()

    cursor.close()
    conn.close()

PostgreSQL Setup (SQL)


CREATE SCHEMA raw;
CREATE SCHEMA curated;
CREATE SCHEMA mart;

-- Raw Layer Table
CREATE TABLE raw.customer_interactions (
    timestamp TIMESTAMP,
    user_id VARCHAR(50),
    query VARCHAR(1000),
    response VARCHAR(1000),
    sentiment VARCHAR(20)
);

-- Curated Layer Table
CREATE TABLE curated.customer_interactions_clean (
    timestamp TIMESTAMP,
    user_id VARCHAR(50),
    query VARCHAR(1000),
    response VARCHAR(1000),
    sentiment_score FLOAT
);

-- Mart Layer Table
CREATE TABLE mart.customer_sentiment_agg (
    hour TIMESTAMP,
    avg_sentiment_score FLOAT,
    query_count INT
);

PostgreSQL Query: Populate Mart Layer


INSERT INTO mart.customer_sentiment_agg
SELECT 
    DATE_TRUNC('hour', timestamp) AS hour,
    AVG(sentiment_score) AS avg_sentiment_score,
    COUNT(*) AS query_count
FROM curated.customer_interactions_clean
GROUP BY DATE_TRUNC('hour', timestamp);

Scalability Strategies

Ensure the AI customer support agent scales with growing demand:

  • Horizontal Scaling: Deploy Rasa and Python microservices on Kubernetes for load balancing.
  • Kafka Partitioning: Use multiple Kafka partitions to handle high-volume customer interactions.
  • Database Optimization: Index PostgreSQL tables and use read replicas for analytics queries.
  • Cloud Flexibility: Deploy on any cloud (AWS, GCP, Azure) or on-premises with Docker.
  • Caching: Use Redis for caching frequent queries to reduce chatbot latency.

These strategies ensure the system handles thousands of concurrent users and large datasets.
